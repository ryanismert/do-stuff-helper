# W15: User Profile Builder

## Objective

Build a skill that conducts an AI-guided life inventory interview based on the 8760 Hours methodology, producing a structured user profile that advisory agents and forward motion tools can reference to align activities with the user's life goals.

## Done When

- A `user-profile-builder` skill exists that conducts a structured interview across twelve life areas
- The interview methodology is pre-distilled from the 8760 Hours PDF into a compact reference guide that the skill reads (not embedded in the prompt)
- The profile covers: present reality snapshot, ideal future vision, major goals, and a self-assessment of skills/strengths/habits
- The profile is stored in a shared location accessible to any activity or agent (`~/exoselfai/docs/user-profile.json`)
- The skill supports two modes: **full interview** (initial build) and **update** (review and revise specific areas)
- The discover skill checks for the profile when starting a new activity and offers to build or update it
- The profile can be regenerated or updated at any time by invoking the skill directly

## Design

### Architecture

```
skills/user-profile-builder/
├── SKILL.md                          # Thin orchestration skill
└── references/
    └── interview-guide.md            # Pre-distilled 8760 Hours methodology
```

**Interview guide** (`references/interview-guide.md`): A structured document distilled from the 8760 Hours PDF. Contains:
- The twelve life areas with descriptions
- Per-area interview questions (adapted from the PDF's review prompts)
- Hamming questions (bottleneck/highest-impact prompts)
- Rating scale guidance (1-7 Likert)
- Ideal future prompts per area
- An additional section on skills, strengths, and work habits (not in the original methodology — see Gaps below)

This file is created once as a task during implementation. The skill reads it at interview time. The raw PDF is not loaded into context.

**Profile output** (`~/exoselfai/docs/user-profile.json`): The structured result of the interview. See Schema below.

### Source Material

The interview methodology is adapted from **8760 Hours v2** by Alex Vermeer (`vault/library/8760-hours-v2.pdf`). The PDF uses mind maps extensively; we adapt the branching exploration into progressive-depth conversational interviewing. The optimization/tracking sections of 8760 Hours are out of scope for this waypoint — those belong to w12 (Forward Motion Analyst).

### Interview Flow

The skill adapts the 8760 Hours four-phase process into a conversational interview:

**Phase 1: Initial Overview** (warm-up)
- Quick brainstorm questions: what went well recently, what didn't, where you tried hard, where you didn't
- Sets the tone and gets the user thinking broadly
- Light touch — 5-10 minutes

**Phase 2: Present Reality** (deep dive per area)
- For each of the twelve life areas:
  - Brief description of what this area covers
  - Open-ended prompt: "How would you describe the current state of [area]?"
  - Follow-up probes for depth (adapted from the PDF's per-area questions)
  - **Progressive depth** — instead of mind maps, the interviewer asks "Can you give me some examples?" or "Would you like to add any detail on [subtopic]?" to expand branches conversationally
  - Likert rating (1-7): "How would you rate this area overall?"
  - Hamming question: "What's the single biggest bottleneck or most important problem in this area?"
- The interviewer should be warm but direct, and not rush — this phase can span multiple sessions

**Phase 3: Ideal Future + Goals**
- For each area: "What would this look like if it were ideal?"
- Yearly theme: "If you had to pick a theme for the next year, what would it be?"
- Focus areas: which areas matter most right now? (rate importance 1-7)
- Extract 3-5 major goals for the next year
- For each goal: what does success look like? How would you measure it?

**Phase 4: Skills, Strengths & Habits** (supplementary — not in 8760 Hours)
- Work habits: how do you work best? When are you most productive? What patterns help or hurt?
- Known tendencies: procrastination patterns, avoidance behaviors, what you gravitate toward vs. avoid
- Key strengths: what are you genuinely good at?
- Skills inventory: technical skills, soft skills, domain expertise
- Learning style: how do you best absorb new information?

This phase fills the gap the user identified — the 8760 Hours methodology doesn't cover an adequate snapshot of skills, strengths, and work habits. This data is particularly valuable for advisory agents that need to understand *how* the user works, not just *what* they want.

### Progressive Depth (Replacing Mind Maps)

The 8760 Hours methodology uses mind maps extensively for branching exploration. Since we're conducting a conversational interview rather than a visual exercise, we replace mind maps with **progressive depth prompting**:

1. Start with a broad open-ended question
2. The interviewer identifies key themes in the response
3. For each theme, offer to go deeper: "You mentioned [X] — would you like to expand on that, or shall we move on?"
4. Continue branching until the user signals they're done with that area
5. Summarize what was captured before moving to the next area

This preserves the mind-mapping benefit (non-linear exploration, catching things you'd miss with linear questions) while working in a conversational format.

### Two Modes

**Full Interview Mode** — Used when no profile exists or the user wants to start fresh. Runs through all four phases. Can be done in one long session or spread across multiple sessions (the skill saves progress to a draft file between sessions).

**Update Mode** — Used when a profile already exists. The skill:
1. Loads the existing profile
2. Asks what triggered the update (new activity, periodic review, life change, etc.)
3. Shows a summary of current state per area
4. Asks which areas to review/update
5. Runs the interview for selected areas only
6. Merges updates into the existing profile, preserving history (previous ratings archived with timestamps)

### Session Continuity

A full interview is long — potentially 1-2 hours spread across sessions. The skill needs to handle interruptions:

- **Draft file**: `~/exoselfai/docs/user-profile-draft.json` stores in-progress interview state
- On start, check for an existing draft and offer to resume
- Each completed area is saved to the draft immediately
- When all phases are complete, the draft is promoted to the final profile and the draft file is deleted

### Discover Skill Integration

Add a step early in the discover skill flow (after Step 1, before the main interview):

1. Check if `~/exoselfai/docs/user-profile.json` exists
2. **If no profile exists**:
   > "Before we dive in, I notice you don't have a life profile yet. Having one would help me understand how this activity fits into your broader goals. Would you like to build one now, or continue without it?"
   - If yes: invoke `do-stuff-helper:user-profile-builder` in full mode, then return to discover
   - If no: continue normally
3. **If profile exists**:
   > "I can see your life profile. After we define this activity, I'll ask how it connects to your broader goals and whether the profile needs updating."
   - After the discover interview completes, ask:
     - "How does [activity] relate to your life goals?"
     - "Does anything in your profile need updating based on this new activity?"
   - If updates needed: invoke `do-stuff-helper:user-profile-builder` in update mode

### What's NOT in Scope

- Optimization, tracking, or course-correction based on the profile (that's w12 Forward Motion Analyst)
- Priority alignment analysis (that's w14)
- Advisory agent conversations (that's w16)
- Procrastination equation, calendar planning, ongoing reviews (w12 territory)
- Personality typing frameworks (e.g., MBTI, Big Five) — could be added later as a profile extension

### Edge Cases

- **User declines full interview**: The skill should produce whatever partial profile was captured. A partial profile is better than none.
- **User wants to skip areas**: Fine — leave those areas empty with a note. The profile doesn't need to be complete to be useful.
- **Session interrupted mid-interview**: Draft file preserves progress. Resume on next invocation.
- **Profile conflicts with activity goals**: Not this skill's job to resolve — surface it for the user and let advisory agents (w16) address it.
- **Multiple users**: Not supported. The profile is for the single exoself user.
