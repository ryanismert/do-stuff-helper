# W20: Plugin Update Workflow

## Objective

Provide a frictionless way to publish and install do-stuff-helper plugin updates after making changes.

## Done When

- A slash command automates the full workflow: version bump, commit, push, marketplace update, plugin reinstall
- The user only needs to start a new conversation afterward

## Design

A **slash command** (`commands/publish.md`) that:

1. Accepts an optional version bump argument (patch/minor/major, defaults to patch)
2. Bumps the version in both `plugin.json` and `marketplace.json`
3. Stages, commits with `chore: bump version to X.Y.Z`, and pushes
4. Runs `claude plugin marketplace update do-stuff-helper`
5. Runs `claude plugin update do-stuff-helper@do-stuff-helper`
6. Tells the user to start a new conversation (skills aren't hotloaded)

### Usage

- `/publish` — bumps patch (0.5.0 → 0.5.1)
- `/publish minor` — bumps minor (0.5.0 → 0.6.0)
- `/publish major` — bumps major (0.5.0 → 1.0.0)

### Implementation Notes

- Use `allowed-tools: ["Bash", "Read", "Edit"]` in the command frontmatter
- Version bump: parse current version from `plugin.json`, increment the appropriate segment
- Steps 4-5 are CLI commands that run non-interactively via Bash
