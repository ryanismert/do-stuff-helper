# W30: Worker Verification Step

## Objective

Add a self-verification step to the worker prompt so workers check their output against the task's acceptance criteria before declaring completion.

## Done When

- The worker prompt template includes a "Verify Before Finishing" section that instructs workers to check each acceptance criterion in the task description and report pass/fail
- The "When Done" section requires workers to include a verification checklist in their summary
- Step 6 (Collect Results) checks for failed/missing verification items and flags them instead of auto-completing
- Backlog item added for a separate verification agent if quality remains a concern

## Design

Three changes to SKILL.md:

### Change 1: Add "Verify Before Finishing" section to worker prompt

Insert after "Discovered Work" and before "When Done" in the worker prompt template.

### Change 2: Update "When Done" section in worker prompt

Add verification checklist to the report format.

### Change 3: Update Step 6 success handling

Check for "NEEDS REVIEW" items — present to user instead of auto-completing.
