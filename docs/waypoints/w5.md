# W5: Autonomy Model

## Objective

Maximize worker agents' ability to self-serve by ensuring they know what capabilities are available and how to leverage them creatively.

## Done When

- Activity CLAUDE.md files include a capabilities section listing MCP servers, skills, project scripts, and environment tools
- The organize skill seeds this section for new activities
- Workers are instructed to self-unblock using available capabilities — including writing and executing ad-hoc code — before escalating

## Design

### Capabilities Section in CLAUDE.md

The organize skill extends the `## do-stuff-helper` section in CLAUDE.md with a `### Capabilities` subsection:

```markdown
### Capabilities

Workers and agents in this project have access to:

**Skills:** organize, discover, research, roadmap, waypoint-design, waypoint-planner, waypoint-implement

**MCP Servers:** <list configured servers and what they do>

**CLI Tools:** <key tools on PATH — e.g., gh, node, docker, bun>

**Project Scripts:** <e.g., npm scripts, Makefile targets>
```

The organize skill populates this dynamically at setup time by checking MCP config, PATH, and package.json/Makefile. Users can edit it afterward as the project evolves.

### Worker Prompt Updates

Update waypoint-implement's worker prompt to add a capabilities section:

```
## Capabilities

Read CLAUDE.md for available skills, MCP servers, and project tools.

You can also write and execute code on the fly. If a task would benefit
from a script — data processing, API calls, file transformations, testing,
automation — write it and run it via Bash. Use whatever language fits the
problem (Python, Node, shell, etc.). Clean up temporary scripts when done.

Before flagging a QUESTION, exhaust your options:
- Check if an available MCP server, skill, or CLI tool solves the problem
- Use WebSearch/WebFetch for research
- Write a script to automate or solve the problem
- Only escalate when truly blocked
```

### Implementation Notes

- The organize skill's Step 5 (CLAUDE.md initialization) gains a capabilities discovery step: check `.mcp.json` / `~/.claude.json` for MCP servers, check PATH for key CLIs, check for package.json/Makefile
- The canonical CLAUDE.md template in the organize skill grows a `### Capabilities` subsection
- waypoint-implement's worker prompt template adds the capabilities and self-unblocking instructions
- No runtime overhead at dispatch — everything is in CLAUDE.md already
