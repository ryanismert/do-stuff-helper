# W18: Home Server Browser Automation Setup

## Objective

Install and configure a virtual display, VNC server, Google Chrome, and the Claude in Chrome extension on the N100 home server so that Claude Code agents can control a browser and the user can remotely observe or intervene.

## Done When

- Xvfb runs a virtual display at a fixed resolution (e.g., 1920x1080)
- x11vnc exposes that display over VNC with password authentication
- Google Chrome is installed and launches against the virtual display
- Claude in Chrome extension is installed in Chrome
- `claude --chrome` successfully connects to and controls Chrome
- All services auto-start on boot via systemd
- User can VNC in from their laptop to see Chrome and take over when needed
- Setup is documented (what was installed, how to connect, how to restart)

### Components

**1. Xvfb (X Virtual Framebuffer)**
- Creates a virtual X11 display in memory (no GPU, no monitor)
- Display number: `:99` (avoids conflict with any real display)
- Resolution: `1920x1080x24` (standard HD, 24-bit color)
- Package: `xvfb`

**2. x11vnc (VNC Server)**
- Attaches to the Xvfb display and serves it over VNC protocol
- Listens on port `5900`
- Password-protected via `-rfbauth` flag (password stored in `~/.vnc/passwd`)
- Bind to `0.0.0.0` on the home network (the N100 is not internet-exposed)
- Package: `x11vnc`

**3. Google Chrome**
- Full Chrome browser (not Chromium — needed for the Claude extension)
- Installed from Google's official apt repository
- Launched with `--display=:99` to render on the virtual display
- No special flags needed — runs in normal headed mode against Xvfb

**4. Claude in Chrome Extension**
- Installed manually on first setup: user VNCs in, opens Chrome, installs from Chrome Web Store
- After installation, the extension persists in the Chrome profile
- Native messaging host configured by `claude --chrome` on first run

### Systemd Services

Three systemd user services, started in dependency order:

**`xvfb.service`** — starts the virtual display
```
ExecStart=/usr/bin/Xvfb :99 -screen 0 1920x1080x24 -ac
```

**`x11vnc.service`** (after xvfb) — starts VNC server
```
ExecStart=/usr/bin/x11vnc -display :99 -rfbauth %h/.vnc/passwd -forever -shared
```

**`chrome-browser.service`** (after xvfb) — starts Chrome
```
Environment=DISPLAY=:99
ExecStart=/usr/bin/google-chrome --no-first-run --disable-session-crashed-bubble
```

All three are `Type=simple` user services (`systemctl --user`) so they don't need root after initial package installation.

### Environment

Claude Code needs `DISPLAY=:99` set when using `--chrome`. Options:
- Set `DISPLAY=:99` in the user's shell profile (`~/.bashrc` or `~/.profile`)
- Or set it in the claude-webhook systemd service environment
- Or pass it per-invocation

Setting it in the shell profile is simplest — it applies everywhere.

### VNC Access

User connects from their laptop using any VNC client:
- **macOS**: Built-in Screen Sharing, or RealVNC Viewer
- **iOS**: RealVNC Viewer, Screens
- **Address**: `<n100-ip>:5900`

### Security

- VNC password authentication (set during setup)
- The N100 is on the home network behind a router, not internet-exposed
- No SSL/TLS needed for VNC on the LAN (could add later if desired)
- Chrome runs as the regular user, not root

## Implementation Notes

### Installation sequence (requires sudo)

1. `sudo apt-get install -y xvfb x11vnc`
2. Install Chrome from Google's apt repo (standard `wget` + `dpkg` method)
3. Create VNC password: `mkdir -p ~/.vnc && x11vnc -storepasswd ~/.vnc/passwd`
4. Create and enable the three systemd user services
5. Set `DISPLAY=:99` in `~/.bashrc`
6. Start services, open VNC, install Chrome extension manually
7. Run `claude --chrome` to set up native messaging host
8. Verify end-to-end: Claude navigates to a page, user sees it via VNC

### What needs user interaction

- **sudo password** — for apt-get install (packages)
- **VNC password creation** — interactive prompt to set password
- **Chrome extension install** — user VNCs in and clicks "Add to Chrome" once
- Everything else can be scripted

### Playwright (deferred)

Playwright + headless Chromium is already partially installed (binaries downloaded, OS deps missing). This can be revisited later as a complementary tool for fully autonomous scraping that doesn't need human intervention. Not needed for this waypoint.
